## 📦 一键部署步骤

### 方法1：使用自动脚本（推荐）

#### 首次部署

1. **配置 Git（仅首次需要）**
   
   打开命令行，运行：
   ```bash
   git config --global user.name "你的名字"
   git config --global user.email "你的邮箱@example.com"
   ```

2. **运行部署脚本**
   
   双击运行：
   ```
   gitee-deploy.bat
   ```

3. **输入 Gitee 账号密码**
   - 首次推送会要求登录
   - 输入您的 Gitee 用户名和密码
   - 可以勾选"记住密码"，下次就不用输入了

4. **启动 Gitee Pages**
   - 访问：https://gitee.com/xh_qianxun/math_game
   - 点击"服务" → "Gitee Pages"
   - 点击"启动"
   - 等待几秒，获得访问链接

#### 后续更新

修改游戏后，只需：
1. 双击 `gitee-deploy.bat`
2. 等待完成
3. 访问 Gitee Pages 设置，点击"更新"
4. 刷新游戏页面

---

### 方法2：手动上传（备用）

如果自动脚本遇到问题：

1. 双击运行 `build.bat` 构建游戏
2. 打开 `dist` 文件夹
3. 全选所有文件（Ctrl+A）
4. 访问 https://gitee.com/xh_qianxun/math_game
5. 点击"上传文件"
6. 拖拽所有文件到网页
7. 选择"覆盖同名文件"
8. 点击"提交"
9. 进入"Gitee Pages"设置，点击"更新"

## ⚠️ 常见问题

### Q1: 运行 gitee-deploy.bat 提示"git 不是内部或外部命令"

**解决方法：**
1. 下载安装 Git：https://git-scm.com/download/win
2. 安装时全部选择默认选项
3. 重启电脑
4. 重新运行脚本

### Q2: 推送时要求输入用户名密码

**正常现象！** 首次推送需要登录：
- 用户名：您的 Gitee 用户名（不是邮箱）
- 密码：您的 Gitee 密码
- 勾选"记住密码"，下次不用再输

### Q3: 推送失败，提示权限问题

**解决方法：**
1. 确保您有仓库的写入权限
2. 在 Gitee 仓库设置中检查是否是管理员
3. 或使用"方法2：手动上传"

### Q4: Gitee Pages 更新后，网页没变化

**解决方法：**
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 强制刷新（Ctrl+F5）
3. 等待1-2分钟让 CDN 更新

### Q5: 部署后打开是空白页

**检查步骤：**
1. 确保上传了 dist 文件夹的**内容**（不是文件夹本身）
2. 确保 index.html 在根目录
3. 在 Gitee Pages 设置中，部署目录选择 "/"
